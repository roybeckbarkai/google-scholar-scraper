Run it locally

If the online version gets blocked by Google Scholar, the easiest fix is to run the app on your own computer.

Download Repository ZIP Open Project Folder Install Node.js Back to app

Windows

  1. Click Download Repository ZIP.
  2. Extract the ZIP to a folder on your computer.
  3. Open the extracted repository folder, then open the inner google-scholar-scraper folder.
  4. If Node.js is not installed yet, click Install Node.js and install the LTS version.
  5. Open the scripts folder.
  6. Double-click start-local.bat.

macOS

  1. Click Download Repository ZIP.
  2. Extract the ZIP to a folder on your computer.
  3. Open the extracted repository folder, then open the inner google-scholar-scraper folder.
  4. If Node.js is not installed yet, click Install Node.js and install the LTS version.
  5. Open the scripts folder.
  6. Double-click start-local.command.

Linux

  1. Click Download Repository ZIP.
  2. Extract the ZIP to a folder on your computer.
  3. Open the extracted repository folder, then open the inner google-scholar-scraper folder.
  4. If Node.js is not installed yet, click Install Node.js and install it.
  5. Open the scripts folder.
  6. Double-click start-local.desktop.
The launcher installs project dependencies automatically on the first run and then starts the full local app. Keep the opened window open while you use the app.
The hosted Vercel app cannot automatically use each visitor's own proxy settings. The scraper runs on Vercel's network, not on the visitor's computer, so local run is the reliable per-user fallback.
On Linux, some file managers ask you to confirm or enable launching desktop files the first time. If you see that prompt, choose the option that allows running start-local.desktop.
