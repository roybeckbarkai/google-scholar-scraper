Local Installation

Run the scraper on your own computer

Local use is recommended when the online Vercel deployment is blocked by Google Scholar with a 403 error. Running locally sends the scraping request from your own machine instead of from a shared serverless IP.

Back to app Download Node.js

Step 1: Download the project

You need the project files on your computer before you can install or run anything.

  1. If the project is on GitHub, open the repository page.
  2. Click Code and then Download ZIP.
  3. Extract the ZIP to a folder on your computer.
  4. Open that extracted folder in your terminal.

If you already use Git, you can clone the repository instead:

git clone <repository-url>
cd google-scholar-scraper

Step 2: Install Node.js

The app runs with Node.js. Installing Node.js also installs npm, which is used to download the app dependencies.

  1. Download Node.js LTS from nodejs.org.
  2. Install it with the default options.
  3. Close and reopen your terminal after installation.
node -v
npm -v

Windows

  1. Open Command Prompt or PowerShell.
  2. Go to the downloaded project folder.
  3. Run the commands below.
scripts\install-local.bat
scripts\run-local.bat

macOS

  1. Open Terminal.
  2. Go to the downloaded project folder.
  3. Run the commands below.
chmod +x scripts/*.sh
./scripts/install-local.sh
./scripts/run-local.sh

Linux

  1. Open a terminal.
  2. Go to the downloaded project folder.
  3. Run the commands below.
chmod +x scripts/*.sh
./scripts/install-local.sh
./scripts/run-local.sh

Manual commands

If you do not want to use the helper scripts, these commands do the same job on any platform:

npm install
npx vercel dev
The app will start with both the frontend and the API routes enabled. That is the mode required for /api/scrape and /api/doi to work.
The online Vercel app cannot automatically detect and use each visitor's personal proxy settings. Browser and operating system proxy settings are local to the user's machine, and Vercel serverless functions run on Vercel's network, not on the visitor's computer. For that reason, local installation is the practical per-user workaround when Google Scholar blocks the hosted deployment.
